# Golden Bird CRM v7.5 Deployment

Manual deploy target: Railway atau Render. Branch kandidat aman: `ByCODE`.

## Environment

Gunakan `.env.example` sebagai dasar dan jangan commit `.env` asli.

Minimum env production:

```dotenv
APP_NAME="Golden Bird CRM"
APP_ENV=production
APP_KEY=base64:GENERATE_ON_PLATFORM
APP_DEBUG=false
APP_URL=https://your-app.example.com

LOG_CHANNEL=stderr
LOG_LEVEL=info

DB_CONNECTION=mysql
DB_HOST=your-db-host
DB_PORT=3306
DB_DATABASE=golden_bird_crm
DB_USERNAME=your-db-user
DB_PASSWORD=your-db-password

CACHE_DRIVER=file
SESSION_DRIVER=file
QUEUE_CONNECTION=sync
FILESYSTEM_DISK=local
MAIL_MAILER=log
VITE_APP_NAME="Golden Bird CRM"
```

## Build Commands

```bash
npm ci
npm run build
composer install --no-dev --optimize-autoloader
php artisan config:cache
php artisan view:cache
```

Do not run `php artisan route:cache` yet. This repo still has a closure route for subscription billing, so route caching is intentionally avoided.

## Start Command

```bash
php artisan migrate --force && php artisan serve --host=0.0.0.0 --port=$PORT
```

Seed demo data manually only when needed:

```bash
php artisan db:seed --force
```

## Railway

1. Create a Railway project from GitHub.
2. Select branch `ByCODE`.
3. Add MySQL or PostgreSQL. If using PostgreSQL, update `DB_CONNECTION=pgsql` and use the matching host, port, database, username, and password.
4. Use Nixpacks. This repo includes `nixpacks.toml` with Node 22, PHP 8.4 packages, `npm ci`, `npm run build`, and Laravel cache warmup.
5. Set the env vars above in Railway Variables.
6. Deploy, then run one-time demo seed from Railway shell if needed:

```bash
php artisan db:seed --force
```

Health check path: `/up`.

## Render

The repo includes `render.yaml`.

1. Create a Blueprint from GitHub.
2. Select branch `ByCODE`.
3. Confirm the web service and database are created in the same region.
4. Confirm `APP_KEY` is generated by Render.
5. Deploy.
6. Run demo seed once from the Render shell if the database is empty:

```bash
php artisan db:seed --force
```

Health check path: `/up`.

## Storage And Cache

If uploads are introduced later, run:

```bash
php artisan storage:link
```

Useful cache reset commands:

```bash
php artisan optimize:clear
php artisan config:cache
php artisan view:cache
```

## Troubleshooting

- Blank or unstyled UI: confirm `npm run build` completed and `public/build/manifest.json` exists on the server.
- 403 after login: verify the user role matches the route middleware in `routes/web.php`.
- 419 CSRF on Render: verify proxy headers remain configured in `bootstrap/app.php`.
- Health check failing: use `/up`, not `/login` or `/`.
- Route cache failure: expected until the subscription billing closure route is converted to a controller action.
